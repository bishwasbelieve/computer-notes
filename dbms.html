DBMS Notes

  • DBMS stands for Database Management System.
  • It is software that allows users to define, create, maintain, and control access to databases.
  • Examples: MySQL, Oracle, SQL Server, PostgreSQL, MongoDB.
  • Main functions:
    • Data storage, retrieval, and update
    • User administration and security
    • Backup and recovery
    • Data integrity management
  • Types of DBMS:
    • Hierarchical
    • Network
    • Relational
    • Object-oriented
  • Advantages: Reduces data redundancy, improves data security, supports multi-user access.

Database and its Purpose

A database is an organized collection of related data. Its purpose is to efficiently store, retrieve, and manage data.

Terminologies in Database

  • Table: Collection of related data in rows and columns.
  • Field: A column in a table; represents a data attribute.
  • Record: A row in a table; represents a single data item.
  • Tuple: Another term for a record (row).
  • Object: An entity with data and behavior (in object-oriented databases).
  • Keys: Attributes used to uniquely identify records (e.g., primary key, foreign key).

Data Dictionary

A data dictionary is a repository that stores definitions and descriptions of data elements in the database.

Database Management System


Introduction

A DBMS is software for creating and managing databases.

Objectives

  • Efficient data storage and retrieval
  • Data security and integrity
  • Support for multi-user environment

Advantages

  • Reduces data redundancy
  • Improves data sharing and security
  • Ensures data consistency

Disadvantages

  • Complexity
  • High cost of hardware and software
  • Requires skilled personnel

Types of Database Model

  • Hierarchical: Data organized in a tree-like structure.
  • Network: Data organized in a graph, allowing many-to-many relationships.
  • Relational: Data organized in tables with relationships between them.
  • Entity Relational: Uses ER diagrams to model data and relationships.

Integrity Constraints and Types

  • Domain Constraint: Restricts the data type and values for a field.
  • Entity Constraint: Ensures each table has a primary key and unique records.
  • Referential Constraint: Ensures foreign keys correctly reference primary keys in related tables.
  • Keys: Used to uniquely identify records and establish relationships.

Normalization

Introduction

Normalization is the process of organizing data in a database to reduce redundancy and improve data integrity.

Normal Forms

  • 1NF (First Normal Form): Ensures each column contains atomic values and each record is unique.
  • 2NF (Second Normal Form): Achieved when it is in 1NF and all non-key attributes are fully functionally dependent on the primary key.
  • 3NF (Third Normal Form): Achieved when it is in 2NF and all attributes are only dependent on the primary key.

Advantages

  • Reduces data redundancy
  • Improves data consistency
  • Makes maintenance easier

Disadvantages

  • Can lead to complex queries
  • May require more joins, affecting performance

Centralized and Distributed Database

Introduction

A centralized database stores all data in a single location, while a distributed database stores data across multiple sites connected by a network.

Advantages

  • Centralized: Easy to manage, better data consistency, simple backup and recovery.
  • Distributed: Improved reliability, faster local access, scalable.

Disadvantages

  • Centralized: Single point of failure, potential bottlenecks.
  • Distributed: Complex management, data synchronization issues.

Comparison

Feature Centralized Distributed
Location Single site Multiple sites
Reliability Lower Higher
Complexity Lower Higher
Performance May be slower for remote users Faster local access

Database Security

Introduction

Database security involves protecting the database from unauthorized access, misuse, or threats.

Challenges

  • Unauthorized access
  • Data breaches
  • SQL injection attacks
  • Data loss or corruption

Security Measures

  • User authentication and authorization
  • Encryption of data
  • Regular backups
  • Auditing and monitoring

Roles of DBA

  • Database installation and configuration
  • User management and security
  • Backup and recovery
  • Performance tuning

Practical Topics

DDL and DML Languages

  • DDL (Data Definition Language): Used to define and modify database structure (e.g., CREATE, ALTER, DROP).
  • DML (Data Manipulation Language): Used to manage data within tables (e.g., SELECT, INSERT, UPDATE, DELETE).

SQL Data Types

  • CHAR: Fixed-length character string.
  • VARCHAR: Variable-length character string.
  • BINARY: Fixed-length binary data.
  • VARBINARY: Variable-length binary data.
  • TINYBLOB: Very small binary large object.
  • TINYTEXT: Very small text string.
  • TEXT: Large text string.
  • LONGTEXT: Very large text string.
  • ENUM: String object with a value chosen from a list of permitted values.
  • BIT: Bit-field type.
  • TINYINT: Very small integer.
  • BOOLEAN: Stores TRUE or FALSE values.
  • INTEGER: Whole number.
  • FLOAT: Single-precision floating-point number.
  • DOUBLE: Double-precision floating-point number.
  • DECIMAL: Fixed-point number.
  • DATE: Date value (YYYY-MM-DD).
  • DATETIME: Date and time value (YYYY-MM-DD HH:MM:SS).

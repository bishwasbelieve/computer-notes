DBMS Notes

  • DBMS stands for Database Management System.
  • It is software that allows users to define, create, maintain, and control access to databases. DBMS acts as an interface between the database and end-users or application programs, ensuring that data is consistently organized and remains easily accessible.
  • Examples: MySQL, Oracle, SQL Server, PostgreSQL, MongoDB, IBM DB2, SQLite, MariaDB.
  • Main functions:
    • Data storage, retrieval, and update: Efficiently stores large amounts of data and allows users to retrieve or modify it using queries.
    • User administration and security: Manages user accounts, roles, and permissions to ensure only authorized users can access or modify data.
    • Backup and recovery: Provides mechanisms to back up data and restore it in case of hardware failure, data corruption, or accidental deletion.
    • Data integrity management: Enforces rules to maintain accuracy and consistency of data, such as constraints and validation checks.
    • Concurrency control: Manages simultaneous data access by multiple users, preventing conflicts and ensuring data consistency.
  • Types of DBMS:
    • Hierarchical: Organizes data in a tree-like structure with parent-child relationships.
    • Network: Allows more complex relationships with a graph structure, supporting many-to-many links.
    • Relational: Uses tables (relations) to store data, with relationships defined by keys. Most widely used model.
    • Object-oriented: Stores data as objects, similar to object-oriented programming concepts.
    • Document-oriented: Stores, retrieves, and manages document-based information, e.g., MongoDB.
  • Advantages: Reduces data redundancy, improves data security, supports multi-user access, provides data abstraction, and ensures data independence.

Data

Raw facts and figures without context, such as numbers, names, dates, or measurements. Data can be qualitative (descriptive, e.g., colors, labels) or quantitative (numeric, e.g., scores, amounts). By itself, data does not provide meaning or insight until it is processed, organized, or analyzed.

  • Types of Data:
    • Structured Data: Organized in a predefined format, such as tables or spreadsheets (e.g., databases, Excel files).
    • Unstructured Data: Lacks a specific format or structure (e.g., emails, images, videos, social media posts).
    • Semi-structured Data: Contains both structured and unstructured elements (e.g., XML, JSON files).
  • Examples: Student names, exam scores, dates of birth, product prices, sensor readings.
  • Characteristics: Data can be accurate or inaccurate, complete or incomplete, relevant or irrelevant, and timely or outdated.

Data is the foundation for generating information, which is achieved by processing, analyzing, and interpreting data to add context and meaning. Data alone has no meaning until it is organized or processed to become useful information. For example, a list of numbers is just data, but when those numbers represent monthly sales and are summarized, they become information that can guide business decisions.

Information

Processed data that is meaningful and useful for decision-making. Information is created by organizing, structuring, or presenting data in a given context so that it becomes valuable to the user. For example, sales data processed to show monthly revenue trends, or exam scores averaged to determine class performance.

Features of Information

  • Accuracy: Information should be correct and free from errors. Inaccurate information can lead to wrong decisions.
  • Timeliness: Information should be available when needed. Outdated information may not be useful for current decisions.
  • Relevance: Information should be applicable to the current context or problem. Irrelevant information can cause confusion.
  • Completeness: All necessary data should be present. Incomplete information can result in poor decisions.
  • Conciseness: Information should be brief but comprehensive, avoiding unnecessary details while covering all important aspects.
  • Consistency: Information should not contradict itself. Consistent information builds trust and reliability.
  • Accessibility: Information should be easily retrievable and understandable by authorized users.
  • Security: Sensitive information should be protected from unauthorized access.

Database and its Purpose

A database is an organized collection of related data, typically stored and accessed electronically. Its purpose is to efficiently store, retrieve, and manage data, supporting multiple users and applications. Databases help in reducing data redundancy, ensuring data integrity, and providing secure access to data. They allow for quick searching, sorting, updating, and reporting of data, making them essential for modern organizations.

  • Databases support data sharing among multiple users and applications.
  • They enforce data consistency and integrity through rules and constraints.
  • Databases provide backup and recovery mechanisms to protect against data loss.
  • They enable data security by controlling user access and permissions.

Terminologies in Database

  • Table: Collection of related data in rows and columns, also called a relation. Each table represents an entity, such as "Students" or "Products".
  • Field (Attribute/Column): A column in a table; represents a data attribute, e.g., "Name" or "Age". Each field has a specific data type.
  • Record (Row/Tuple): A row in a table; represents a single data item, e.g., a student's details. Each record is a unique instance of the entity.
  • Tuple: Another term for a record (row). In relational databases, a tuple is an ordered set of values.
  • Object: An entity with data and behavior (in object-oriented databases). Objects encapsulate both state (data) and methods (behavior).
  • Keys: Attributes used to uniquely identify records (e.g., primary key, foreign key). Keys establish relationships between tables and ensure data integrity.
  • Primary Key: A field or combination of fields that uniquely identifies each record in a table.
  • Foreign Key: A field in one table that refers to the primary key of another table, establishing a relationship between the two tables.
  • Schema: The structure that defines the organization of data in a database, including tables, fields, relationships, and constraints.
  • Instance: The actual content of the database at a particular moment. The schema is the design, while the instance is the data at a given time.
  • Entity: An object or thing in the real world that is distinguishable from other objects, represented as a table in a database.
  • Attribute: A property or characteristic of an entity, represented as a field in a table.
  • Relationship: An association among two or more entities, often represented by foreign keys.

Data Dictionary

A data dictionary is a repository that stores definitions and descriptions of data elements in the database. It includes information such as data types, allowed values, default values, and relationships between tables. It helps in maintaining consistency and understanding the structure of the database.

Database Management System


Introduction

A DBMS is software for creating and managing databases. It provides tools for defining, constructing, manipulating, and sharing databases among users and applications.

Objectives

  • Efficient data storage and retrieval: Optimizes how data is stored and accessed.
  • Data security and integrity: Protects data from unauthorized access and ensures accuracy.
  • Support for multi-user environment: Allows concurrent access by multiple users without conflicts.
  • Data independence: Changes in data structure do not affect application programs.

Advantages

  • Reduces data redundancy: Eliminates duplicate data by centralizing storage.
  • Improves data sharing and security: Controls access and provides secure sharing among users.
  • Ensures data consistency: Maintains uniformity and accuracy across the database.
  • Facilitates backup and recovery: Simplifies data backup and restoration processes.

Disadvantages

  • Complexity: Requires specialized knowledge to design and manage.
  • High cost of hardware and software: Licensing and infrastructure can be expensive.
  • Requires skilled personnel: Needs trained database administrators and developers.
  • Performance overhead: Additional layers may slow down simple operations compared to file systems.

Types of Database Model

  • Hierarchical: Data organized in a tree-like structure. Each child has only one parent. Example: IBM IMS.
  • Network: Data organized in a graph, allowing many-to-many relationships. Example: Integrated Data Store (IDS).
  • Relational: Data organized in tables with relationships between them. Uses SQL for queries. Example: MySQL, PostgreSQL.
  • Entity Relational: Uses ER diagrams to model data and relationships. ER diagrams help in designing the logical structure of databases.
  • Object-oriented: Data is stored as objects, supporting inheritance, encapsulation, and polymorphism. Example: db4o, ObjectDB.

Integrity Constraints and Types

  • Domain Constraint: Restricts the data type and values for a field. For example, age must be an integer between 0 and 120.
  • Entity Constraint: Ensures each table has a primary key and unique records. No two rows can have the same primary key value.
  • Referential Constraint: Ensures foreign keys correctly reference primary keys in related tables, maintaining relationships between tables.
  • Keys: Used to uniquely identify records and establish relationships. Types include primary key, foreign key, candidate key, and super key.
  • Unique Constraint: Ensures all values in a column are unique.
  • Not Null Constraint: Ensures that a column cannot have NULL values.

Normalization

Introduction

Normalization is the process of organizing data in a database to reduce redundancy and improve data integrity. It involves decomposing tables to eliminate data anomalies and ensure logical data storage.

Normal Forms

  • 1NF (First Normal Form): Ensures each column contains atomic values and each record is unique. No repeating groups or arrays.
  • 2NF (Second Normal Form): Achieved when it is in 1NF and all non-key attributes are fully functionally dependent on the primary key. Eliminates partial dependency.
  • 3NF (Third Normal Form): Achieved when it is in 2NF and all attributes are only dependent on the primary key. Eliminates transitive dependency.
  • BCNF (Boyce-Codd Normal Form): A stricter version of 3NF, every determinant is a candidate key.

Advantages

  • Reduces data redundancy: Prevents duplicate data storage.
  • Improves data consistency: Ensures data is updated in one place.
  • Makes maintenance easier: Simplifies updates and modifications.

Disadvantages

  • Can lead to complex queries: Data spread across multiple tables may require complex joins.
  • May require more joins, affecting performance: Excessive normalization can slow down query performance.

Centralized and Distributed Database

Introduction

A centralized database stores all data in a single location, usually on a central server. A distributed database stores data across multiple sites connected by a network, with each site managing its own data and participating in global queries.

Advantages

  • Centralized: Easy to manage, better data consistency, simple backup and recovery, lower hardware costs.
  • Distributed: Improved reliability, faster local access, scalable, supports data sharing across locations, fault tolerance.

Disadvantages

  • Centralized: Single point of failure, potential bottlenecks, limited scalability, higher risk of data loss.
  • Distributed: Complex management, data synchronization issues, higher setup and maintenance costs, network dependency.

Comparison

Database Security

Introduction

Database security involves protecting the database from unauthorized access, misuse, or threats. It ensures confidentiality, integrity, and availability of data.

Challenges

  • Unauthorized access: Preventing access by users without proper permissions.
  • Data breaches: Protecting sensitive data from being leaked or stolen.
  • SQL injection attacks: Preventing attackers from executing malicious SQL code.
  • Data loss or corruption: Protecting data from accidental or intentional loss or modification.
  • Insider threats: Preventing misuse by authorized users.

Security Measures

  • User authentication and authorization: Verifying user identity and granting appropriate access rights.
  • Encryption of data: Protecting data at rest and in transit using encryption algorithms.
  • Regular backups: Creating copies of data to restore in case of loss or corruption.
  • Auditing and monitoring: Tracking database activities to detect suspicious behavior.
  • Firewalls and network security: Restricting access to the database server.
  • Access control lists (ACLs): Defining which users or roles can access specific data.

Roles of DBA

  • Database installation and configuration: Setting up and tuning the DBMS software.
  • User management and security: Creating user accounts, assigning roles, and managing permissions.
  • Backup and recovery: Planning and executing data backup and restoration.
  • Performance tuning: Optimizing queries, indexing, and system resources for better performance.
  • Monitoring and troubleshooting: Ensuring database health and resolving issues.

Practical Topics

DDL and DML Languages

  • DDL (Data Definition Language): Used to define and modify database structure (e.g., CREATE, ALTER, DROP). DDL commands affect the schema of the database.
  • DML (Data Manipulation Language): Used to manage data within tables (e.g., SELECT, INSERT, UPDATE, DELETE). DML commands affect the data stored in the database.
  • DCL (Data Control Language): Used to control access to data (e.g., GRANT, REVOKE).
  • TCL (Transaction Control Language): Used to manage transactions (e.g., COMMIT, ROLLBACK, SAVEPOINT).

SQL Data Types

  • CHAR: Fixed-length character string. Example: CHAR(10).
  • VARCHAR: Variable-length character string. Example: VARCHAR(255).
  • BINARY: Fixed-length binary data.
  • VARBINARY: Variable-length binary data.
  • TINYBLOB: Very small binary large object, up to 255 bytes.
  • TINYTEXT: Very small text string, up to 255 characters.
  • TEXT: Large text string, up to 65,535 characters.
  • LONGTEXT: Very large text string, up to 4GB.
  • ENUM: String object with a value chosen from a list of permitted values. Example: ENUM('male','female').
  • BIT: Bit-field type, stores binary values.
  • TINYINT: Very small integer, typically 1 byte.
  • BOOLEAN: Stores TRUE or FALSE values (often implemented as TINYINT).
  • INTEGER: Whole number, typically 4 bytes.
  • FLOAT: Single-precision floating-point number.
  • DOUBLE: Double-precision floating-point number.
  • DECIMAL: Fixed-point number, used for precise values like currency.
  • DATE: Date value (YYYY-MM-DD).
  • DATETIME: Date and time value (YYYY-MM-DD HH:MM:SS).
  • TIMESTAMP: Stores date and time, often used for tracking changes.
  • TIME: Stores time values (HH:MM:SS).
